## 原则

- **后端分层**：路由 → Service → Repository → Model，层间单向依赖
- **前端功能优先**：相同功能的组件、composable、类型放在同一 `composables/` 或就近
- `pages/` 只负责布局组装，数据获取委托给 composable 或 store
- `components/` 只放无业务逻辑的通用 UI 组件
- 跨页面状态放 `stores/`，单页面状态用 `ref`/`reactive`

## 文件命名规则

| 类型 | 规范 | 示例 |
|---|---|---|
| Vue 组件 | PascalCase + `.vue` | `QuestionCard.vue` |
| Composable | `useXxx.ts` | `useQuestions.ts` |
| Pinia Store | `useXxxStore.ts` | `useQuestionStore.ts` |
| API 模块 | resource name + `.ts` | `questions.ts` |
| Python 模块 | snake_case | `question_service.py` |
| 测试文件 | 与源文件同名 + `.test.ts` 或 `test_xxx.py` | — |
