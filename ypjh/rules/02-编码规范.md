# 02 - 编码规范

## Python 后端规范

### 类型与语言

- Python 3.12，全量类型注解，启用 mypy strict 模式
- 禁止裸 `except: pass`，捕获异常必须至少记录或重抛
- 使用 `|` 联合类型语法（`str | None`），不用 `Optional[str]`
- 字符串用双引号（Ruff formatter 自动统一）

### 命名规范

| 对象 | 规范 | 示例 |
|------|------|------|
| 模块/文件 | snake_case | `question_service.py` |
| 函数/方法 | snake_case | `get_question_by_id` |
| 类 | PascalCase | `QuestionService` |
| 常量 | SCREAMING_SNAKE_CASE | `CONFIDENCE_THRESHOLD` |
| Pydantic model | PascalCase | `QuestionCreate` |
| 数据库字段 | snake_case | `next_review_at` |

### 分层规则

- **路由**（`api/v1/endpoints/`）：只做参数接收 + 调用 service + 返回响应，禁止业务逻辑
- **Service**（`services/`）：业务逻辑，可调用多个 Repository，负责事务边界
- **Repository**（`repositories/`）：所有 DB 操作，只返回模型或原始数据，不含业务判断
- **禁止**在路由 handler 中直接操作 ORM 模型

### 异步规范

- 数据库操作全部使用 `async/await`（SQLAlchemy async session）
- 禁止在 async 函数中调用阻塞 IO（`requests`、`open()` 等），用 `httpx.AsyncClient` / `aiofiles`
- `session.flush()` 在 Repository 内，`session.commit()` 在 Service 或事务管理器内

## TypeScript / Vue 前端规范

### 类型与语言

- 全量 TypeScript，禁止 `any`，必要时用 `unknown` + 类型守卫
- `tsconfig.json` 启用 `"strict": true`
- 优先 `interface` 定义对象，`type` 用于联合类型

### 命名规范

| 对象 | 规范 | 示例 |
|------|------|------|
| Vue 组件 | PascalCase + `.vue` | `QuestionCard.vue` |
| Composable | `useXxx.ts` | `useQuestions.ts` |
| Pinia Store | `useXxxStore.ts` | `useQuestionStore.ts` |
| API 模块 | resource name | `questions.ts` |
| 函数/变量 | camelCase | `fetchQuestions` |
| 常量 | SCREAMING_SNAKE_CASE | `MAX_DIFFICULTY` |
| CSS 类名 | kebab-case（Tailwind 工具类优先） | — |

### 组件规范

- 强制使用 `<script setup lang="ts">`，禁止 Options API
- 文件字段名与后端保持一致（snake_case：`subject_id`、`next_review_at`）

### 导入规范

```typescript
// 顺序：外部库 → 内部绝对路径（@/） → 相对路径 → 类型
import { ref, computed } from 'vue'
import { useRouter } from 'vue-router'

import { questionsApi } from '@/api/questions'
import { useQuestionStore } from '@/stores/useQuestionStore'

import QuestionCard from './QuestionCard.vue'
import type { Question } from '@/types'
```

- 使用路径别名 `@/` 指向 `src/`
- 禁止循环导入

### 错误处理

- 异步操作统一 `try/catch`
- 用户可见错误通过 Toast 通知展示（不暴露内部错误详情）
- 内部错误 `console.error` 记录（生产环境接入错误监控）
- API 层不 try/catch，错误向上冒泡到 composable/store 层处理
