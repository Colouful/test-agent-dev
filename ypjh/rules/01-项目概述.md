# 01 - 项目概述

## 项目名称

错题本（Wrong Answer Notebook）

## 项目背景

本项目是 **Code Agent Harness 工程实践**课程的练手台架。全班做同一个错题本，目的不是功能最多，而是走完从需求设计到云端部署的完整 harness 工作流，带走一套可复用的工程判断力。

## 核心功能（四大模块）

| 功能 | 说明 | 优先级 |
|------|------|------|
| 拍照识别错题 | 上传图片 → Bedrock 识别题目内容 → 结构化入库 | P0（主线） |
| 错题存储管理 | 按学科/章节/难度分类，支持手动录入和编辑 | P0 |
| 智能复习推荐 | 基于 SM-2 遗忘曲线算法推送待复习错题 | P1 |
| 打印排版 | 生成可打印的错题卡片（排版美观，不用真打印） | P2 |

## 技术栈

### 后端
- **框架**：FastAPI（Python 3.12）
- **包管理**：uv
- **ORM**：SQLAlchemy 2.x（async）
- **数据库**：SQLite（本地开发）/ DynamoDB（AWS 生产）
- **数据校验**：Pydantic v2
- **认证**：JWT（python-jose）
- **测试**：pytest + httpx AsyncClient

### 前端
- **框架**：Vue 3 + TypeScript（`<script setup>` 语法）
- **构建工具**：Vite
- **UI 组件库**：Element Plus 或 Tailwind CSS
- **状态管理**：Pinia
- **路由**：Vue Router v4
- **HTTP 客户端**：Axios
- **测试**：Vitest + Vue Testing Library

### AWS 云服务
- **Bedrock**：图片识别（调用视觉模型识别错题内容）
- **S3**：存储上传的错题图片
- **DynamoDB**：生产环境存储题目数据（表名：`cuotiben-questions`）

### 工具链
- **Ruff**：Python 格式化 + lint
- **mypy**：Python 类型检查
- **Prettier + ESLint**：前端格式化 + lint
- **husky + lint-staged**：Git hooks
- **Playwright**：E2E 测试（V2）

## 数据模型（核心实体）

```python
class Question(Base):
    id: int                    # 主键
    user_id: str               # 用户隔离（MUST 隔离）
    subject_id: int            # 学科
    type: QuestionType         # single / multiple / fill / essay
    content: str               # 题目内容（结构化存储，禁止裸字符串）
    wrong_answer: str | None   # 我的错误答案
    correct_answer: str        # 正确答案
    analysis: str | None       # 解析
    difficulty: int            # 1-5
    confidence_score: float    # Bedrock 识别置信度（缺失按 0 处理）
    ease_factor: float         # SM-2 遗忘曲线系数（初始 2.5）
    review_count: int          # 复习次数
    next_review_at: datetime | None  # 下次复习时间
    image_key: str | None      # S3 图片 key
    created_at: datetime
    updated_at: datetime
```

## 识别流程（主线业务）

```
用户上传图片
    ↓
S3 存储图片（返回 image_key）
    ↓
调用 Bedrock 识别
    ↓
格式校验（必须有 confidence 字段，缺失按 0）
    ↓
confidence >= 阈值 → 直接展示供用户确认
confidence <  阈值 → 标记"待确认"，不直接入库
    ↓
用户确认后入库（结构化 Question 对象）
```

## 关键约束（不可违反）

- **R1**：错题数据必须按 `user_id` 严格隔离，禁止跨用户查询（安全边界）
- **R2**：Bedrock 返回必须先做 schema 格式校验，`confidence` 缺失字段按 0 处理，不得默认为 1.0
- **R3**：题目内容必须结构化存储（`Question` 对象），禁止将题目存为裸字符串入库
- **R4**：低置信度识别结果（< 阈值）不得直接入库，必须标记待确认

## 开发阶段

| 阶段 | 内容 |
|------|------|
| **Day 1**（Harness 搭建）| 规则层 + 技能层 + 工具层，不写业务代码 |
