# 10 - 文档规范

## 注释原则

只在 **WHY 非显而易见** 时写注释：隐藏约束、微妙的不变式、特定 bug 的 workaround、会让读者惊讶的行为。

```typescript
// 坏：解释 what（代码已自述）
// 过滤过期题目
questions.filter(q => ...)

// 好：解释 why（隐藏约束）
// SM-2 要求间隔至少 1 天，避免同天触发两次复习导致权重失真
const interval = Math.max(1, baseInterval * easeFactor)
```

```python
# 好：解释隐藏约束
# flush 而非 commit：事务由调用方（service）统一提交，保证原子性
await session.flush()
```

## 禁止事项

- 禁止多段落 docstring / 多行注释块（一行 max）
- 禁止引用任务编号、PR 号、调用方（`# added for issue #123`、`# used by ReviewPage`）
- 禁止注释掉的死代码，直接删除

## Commit Message 格式

```
<type>(<scope>): <subject>
```

- `type`: `feat` | `fix` | `refactor` | `style` | `test` | `docs` | `chore`
- `scope`: 影响的模块，如 `questions`、`review`、`auth`、`store`、`router`
- `subject`: 动词开头，说明做了什么，不超过 72 字符，不加句号

```
feat(questions): support markdown image upload in question content
fix(review): correct SM-2 ease factor update on rating 1
refactor(api): extract pagination logic to shared dependency
chore(deps): upgrade FastAPI to 0.112
```

中文 subject 同样可接受（团队统一即可）：

```
feat(questions): 支持错题内容上传图片
fix(review): 修正评分为 1 时遗忘曲线系数计算错误
```

## README 规范

项目根目录 `README.md` 包含：

1. 项目简介（1 句话）
2. 功能特性（要点列表）
3. 快速开始（3 步内跑起来）
4. 环境变量说明（`.env.example` 对应）
5. 目录结构（简化版）

不写架构决策、TODO、详细 API 文档（这些在 `rules/` 和 Swagger `/docs` 中）。
