# 07 - 状态管理（Pinia）

## 状态分层原则

| 状态类型 | 存放位置 | 示例 |
|---|---|---|
| 服务器状态（异步） | Composable（`useXxx.ts`）| 错题列表、复习计划 |
| 全局持久状态 | Pinia store | 用户认证、主题设置 |
| 页面级状态 | 页面组件 `ref`/`reactive` | 当前筛选条件、分页 |
| 组件内状态 | 组件内 `ref` | 展开/收起、输入值 |

## Pinia Store 规范

使用 **Setup Store**（Composition API 风格），不用 Options Store：

```typescript
// stores/useQuestionStore.ts
import { ref, computed } from 'vue'
import { defineStore } from 'pinia'
import { questionsApi } from '@/api/questions'
import type { Question } from '@/types'

export const useQuestionStore = defineStore('question', () => {
  // State
  const questions = ref<Question[]>([])
  const loading = ref(false)
  const error = ref<string | null>(null)

  // Getters（computed）
  const overdueQuestions = computed(() =>
    questions.value.filter(
      (q) => q.next_review_at && new Date(q.next_review_at) < new Date(),
    ),
  )

  // Actions
  async function fetchAll(params?: { subject_id?: number }) {
    loading.value = true
    error.value = null
    try {
      const result = await questionsApi.list(params)
      questions.value = result.items
    } catch (e) {
      error.value = (e as Error).message
    } finally {
      loading.value = false
    }
  }

  async function create(data: QuestionCreate) {
    const question = await questionsApi.create(data)
    questions.value.unshift(question)
    return question
  }

  async function remove(id: number) {
    await questionsApi.remove(id)
    questions.value = questions.value.filter((q) => q.id !== id)
  }

  return { questions, loading, error, overdueQuestions, fetchAll, create, remove }
})
```

## Composable 规范（服务器状态）

页面级的异步数据获取用 composable，不放 store（避免全局缓存副作用）：

```typescript
// composables/useQuestions.ts
import { ref, onMounted } from 'vue'
import { questionsApi } from '@/api/questions'
import type { Question, PaginatedResult } from '@/types'

export function useQuestions(subjectId?: number) {
  const data = ref<PaginatedResult<Question> | null>(null)
  const loading = ref(false)
  const error = ref<string | null>(null)

  async function load(page = 1) {
    loading.value = true
    error.value = null
    try {
      data.value = await questionsApi.list({ page, subject_id: subjectId })
    } catch (e) {
      error.value = (e as Error).message
    } finally {
      loading.value = false
    }
  }

  onMounted(() => load())

  return { data, loading, error, load }
}
```

## 规则

- 每个 store 对应一个业务域，命名 `useXxxStore.ts`（如 `useAuthStore`、`useQuestionStore`）
- **服务器数据**放 composable，**全局 UI/认证状态**放 store
- 组件中直接解构 store，不要 `storeToRefs` 包 action（action 不是响应式）：

```typescript
// 好
const store = useQuestionStore()
const { questions, loading } = storeToRefs(store)  // 响应式数据
const { fetchAll, create } = store                  // actions 直接取

// 坏：整个 store 直接解构会丢失响应式
const { questions } = useQuestionStore()  // ❌
```

- 禁止在 store 外直接修改 store 的 state，所有修改通过 action

## 派生状态

优先用 `computed` 在 store 内计算，不在组件模板里写过滤/排序逻辑：

```typescript
// store 内
const hardQuestions = computed(() =>
  questions.value.filter((q) => q.difficulty >= 4),
)
```
