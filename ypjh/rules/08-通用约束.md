# 08 - 通用约束

## 代码质量硬性要求

### 前端

- 禁止提交含 `console.log` 的代码
- 禁止 `any` 类型，禁止 `// @ts-ignore` / `// @ts-nocheck`
- 禁止魔法数字，定义为有名称的常量：

```typescript
// 坏
if (question.difficulty > 3) { ... }

// 好
const HARD_DIFFICULTY_THRESHOLD = 3 as const
if (question.difficulty > HARD_DIFFICULTY_THRESHOLD) { ... }
```

### 后端

- 禁止裸 `except: pass`（必须至少记录或重抛）
- 禁止在 route handler 中直接 `import` ORM 模型操作 DB
- 禁止把 `settings.secret_key` 等敏感字段记录到日志

## 不允许做的事

- **直接修改 props**（Vue）或 Pinia state（在 action 外）
- **在 `<template>` 里直接调接口**（封装到 composable）
- **在组件里硬编码颜色值**（用 CSS 变量或 Tailwind token）
- **把 JWT token 明文写入 `localStorage`**（可用，但不要 log 出来）
- **把密钥/token 写入代码文件**（使用 `.env` 环境变量）
- **使用 `v-html` 渲染用户输入**（必须先用 DOMPurify 净化）

## 可访问性（a11y）

- 交互元素可聚焦、可键盘操作
- 图片有有意义的 `alt` 属性（纯装饰图 `alt=""`）
- 表单控件关联 `<label>` 或 `aria-label`
- 使用语义化 HTML（`<button>` 而非 `<div @click>`，`<nav>` 而非 `<div class="nav">`）

## 安全约束

| 风险 | 规则 |
|---|---|
| XSS | 禁止 `v-html` 渲染用户输入；必须用时先 DOMPurify |
| SQL 注入 | 全部通过 SQLAlchemy ORM 参数化，不拼接 SQL |
| 认证 | JWT 验证在 FastAPI dependency，不在业务代码里检查 |
| 敏感信息泄露 | 错误响应不暴露堆栈、SQL 详情 |
| CORS | 只在 `settings.allowed_origins` 白名单内，不用 `*` |

## 依赖管理

- **后端**：通过 `uv add` 添加依赖，不手动编辑 `pyproject.toml` 依赖列表
- **前端**：锁定版本，使用 `package-lock.json`；定期 `npm audit`
- 引入新包前确认：是否有原生方案？包大小是否合理？维护状态？
