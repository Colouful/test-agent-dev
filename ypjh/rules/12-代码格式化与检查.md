# 12 - 代码格式化与检查

## 工具链总览

| 层 | 工具 | 用途 |
|---|---|---|
| 后端 | **Ruff** | 格式化 + lint（替代 Black + isort + Flake8）|
| 后端 | **mypy** | 静态类型检查 |
| 前端 | **Prettier** | 代码格式化 |
| 前端 | **ESLint** | 代码质量检查（vue + typescript）|
| 前端 | **tsc** | TypeScript 类型检查 |
| 通用 | **lint-staged** | 提交前自动检查 |
| 通用 | **husky** | Git hooks 管理 |

---

## 后端配置

### Ruff（`pyproject.toml`）

```toml
[tool.ruff]
line-length = 100
target-version = "py312"

[tool.ruff.lint]
select = ["E", "F", "I", "UP", "B", "SIM", "TCH"]
ignore = ["E501"]  # 行长由 formatter 管理

[tool.ruff.format]
quote-style = "double"
indent-style = "space"
```

### mypy（`pyproject.toml`）

```toml
[tool.mypy]
python_version = "3.12"
strict = true
plugins = ["pydantic.mypy"]

[[tool.mypy.overrides]]
module = "tests.*"
disallow_untyped_defs = false
```

### 后端 npm scripts（Makefile 风格）

```bash
# pyproject.toml [tool.uv.scripts] 或 Makefile
uv run ruff check .          # lint 检查
uv run ruff check . --fix    # 自动修复
uv run ruff format .         # 格式化
uv run mypy .                # 类型检查
uv run pytest                # 运行测试
```

## 前端配置

### Prettier（`.prettierrc`）

```json
{
  "semi": false,
  "singleQuote": true,
  "tabWidth": 2,
  "trailingComma": "es5",
  "printWidth": 100,
  "plugins": ["prettier-plugin-tailwindcss"]
}
```

### ESLint（`eslint.config.ts`，Flat Config）

```typescript
import pluginVue from 'eslint-plugin-vue'
import tseslint from 'typescript-eslint'

export default tseslint.config(
  ...pluginVue.configs['flat/recommended'],
  ...tseslint.configs.strict,
  {
    rules: {
      'no-console': 'error',
      '@typescript-eslint/no-explicit-any': 'error',
      'vue/component-api-style': ['error', ['script-setup']],
      'vue/define-macros-order': ['error', {
        order: ['defineProps', 'defineEmits', 'defineSlots'],
      }],
      'import/order': ['error', {
        groups: ['builtin', 'external', 'internal', 'parent', 'sibling'],
        'newlines-between': 'always',
      }],
    },
  },
)
```

### NPM Scripts

```bash
npm run lint          # ESLint 检查
npm run lint:fix      # ESLint 自动修复
npm run format        # Prettier 格式化
npm run type-check    # tsc --noEmit（不编译，只检查类型）
```

## Git Hooks（husky + lint-staged）

**pre-commit**：自动格式化 + 检查暂存文件

```json
// package.json
{
  "lint-staged": {
    "frontend/src/**/*.{ts,vue}": ["eslint --fix", "prettier --write"],
    "frontend/src/**/*.{css,json}": "prettier --write",
    "backend/**/*.py": ["uv run ruff check --fix", "uv run ruff format"]
  }
}
```

**commit-msg**：commitlint 校验格式

```js
// commitlint.config.js
export default {
  extends: ['@commitlint/config-conventional'],
  rules: {
    'scope-enum': [2, 'always', [
      'questions', 'review', 'stats', 'subjects', 'auth',
      'store', 'router', 'api', 'db', 'deps', 'ci',
    ]],
  },
}
```

## CI 检查顺序

```yaml
# 每次 PR 必须全部通过
steps:
  - name: 后端类型检查
    run: cd backend && uv run mypy .
  - name: 后端 lint
    run: cd backend && uv run ruff check .
  - name: 后端测试
    run: cd backend && uv run pytest --cov
  - name: 前端类型检查
    run: cd frontend && npm run type-check
  - name: 前端 lint
    run: cd frontend && npm run lint
  - name: 前端测试
    run: cd frontend && npm run test:run
  - name: 前端构建验证
    run: cd frontend && npm run build
```
