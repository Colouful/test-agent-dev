# 04 - 组件规范（Vue 3）

## 基本结构（`<script setup>` 优先）

```vue
<!-- components/QuestionCard.vue -->
<script setup lang="ts">
// 1. 导入
import { computed } from 'vue'
import type { Question } from '@/types'

// 2. Props & Emits
interface Props {
  question: Question
  readonly?: boolean
}

interface Emits {
  edit: [id: number]
  delete: [id: number]
}

const props = withDefaults(defineProps<Props>(), { readonly: false })
const emit = defineEmits<Emits>()

// 3. 响应式状态
const isExpanded = ref(false)

// 4. 计算属性
const difficultyLabel = computed(() => {
  const map: Record<number, string> = { 1: '简单', 2: '较易', 3: '中等', 4: '较难', 5: '困难' }
  return map[props.question.difficulty] ?? '未知'
})

// 5. 事件处理
function handleEdit() {
  emit('edit', props.question.id)
}
</script>

<template>
  <div class="...">
    <slot name="header" />
    <!-- 内容 -->
  </div>
</template>
```

## 规则

- **强制使用 `<script setup lang="ts">`**，不用 Options API
- 每个 `.vue` 文件只包含一个组件
- Props 用 `interface` 定义，Emits 用 `interface` 定义（类型安全事件）
- 组件不超过 **150 行**（`<template>` + `<script>` 合计），超出则拆分子组件
- 禁止在 `<template>` 中写复杂表达式，提取为 `computed`

## 复用层级

| 层级 | 放置位置 | 说明 |
|---|---|---|
| 基础 UI | `components/ui/` | Button、Input、Dialog — 无业务逻辑 |
| 布局 | `components/layout/` | AppHeader、AppSidebar |
| 页面 | `pages/` | 只负责组装，不写业务逻辑 |

## Props 规范

```typescript
// 好：使用 withDefaults 提供默认值
const props = withDefaults(defineProps<Props>(), {
  readonly: false,
  size: 'medium',
})

// 好：v-model 双向绑定
interface Props {
  modelValue: string
}
interface Emits {
  'update:modelValue': [value: string]
}
```

## 性能规则

- 列表 `v-for` 必须绑定 `:key`，使用数据的 `id`，不用 `index`
- 不提前使用 `v-memo`，确认有性能问题再加
- 图片加 `loading="lazy"`
- 大列表考虑虚拟滚动（`vue-virtual-scroller`）

## 模板规范

```vue
<!-- 好：指令顺序 v-if > v-for > v-bind > v-on -->
<li
  v-for="item in list"
  :key="item.id"
  :class="{ active: item.selected }"
  @click="select(item)"
>

<!-- 禁止 v-if 和 v-for 同元素，用 <template v-for> 包装 -->
<template v-for="item in list" :key="item.id">
  <li v-if="item.visible">...</li>
</template>
```
